**Components needed (derived from roadmap phases):**
- **Button**: Primary (orange), secondary (neutral), danger (red), ghost. Multiple sizes.
- **Card**: The #0b0b0b bordered container used everywhere. With optional header/title.
- **TextInput**: Dark-themed input with label, placeholder, error state, focus ring (orange).
- **TextArea**: Same styling as TextInput but multi-line.
- **Checkbox**: Custom dark-themed checkbox with orange check mark.
- **Select**: Dark dropdown matching input styling.
- **Badge**: Status indicators — green (active/signed), yellow (pending), red (rejected/expired), neutral (draft). Used in workflow and dashboard.
- **SectionHeader**: Page section divider with title and optional description.
- **Alert**: Info/success/warning/error banners for form feedback.

**Pattern guidance:**
- All components use Tailwind utility classes directly
- Props for variants (size, color, state) — use simple conditional classNames, NOT a CSS-in-JS library
- Components are in src/components/ui/ directory
- Each component is a single file with named export
- Focus states use ring-orange-500/50 to match accent color
- Disabled states use opacity-50 cursor-not-allowed
- Transitions: transition-all duration-200 for interactive elements
